## 1. Objective
The **Vertical Orchestrator** is responsible for establishing logical bridges across the AIWF's heterogeneous vertical shards. It enables the bidirectional propagation of high-level intelligence (e.g., Legal compliance updates affecting Financial routing) and enforces the **Universal Vertical Ontology**.

## 2. Core Logic & Behavioral Protocols

### 2.1 Cross-Sector Intelligence Bridge
- **Synthesis**: Aggregates metadata from specialized vertical agents (e.g., `legal_auditor`, `hospitality_concierge`).
- **Translation**: Translates sector-specific data into the **Universal Industrial Format** (UIF) for global consumption.
- **Propagation**: Triggers real-time sync events across dependent vertical shards when a core sector variable changes.

### 2.2 Universal Adapter Management
- **Scaffolding**: Automates the creation of vertical-specific adapters that comply with the `spec_vertical_adapter_schema.json`.
- **Normalization**: Ensures all vertical I/O follows strict OMEGA-tier naming and structure conventions.

### 2.3 Sectoral Geofencing
- Ensures that vertical intelligence propagation respects Law 151/2020 geofencing, especially when bridging Finance and Legal sectors.

## 3. Integration Matrix

| Vertical | Interaction Type | Purpose |

## 4. Operational Commands
- `/vertical bridge <source> <target>`: Establishes a permanent intelligence link between sectors.
- `/vertical sync --all`: Reconciles intelligence across all active vertical shards.
- `/vertical adapter generate <sector>`: Scaffolds a new compliant vertical adapter.

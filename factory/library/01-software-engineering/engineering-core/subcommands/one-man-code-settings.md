# One Man Code — Settings Wizard Flow

When user runs **`/man settings`**, run this wizard step by step. One step at a time; confirm before proceeding to next.

**Target file:** `docs/development/learning/ONE_MAN_CODE_SETTINGS.md`. Create it if missing (use template below).

---

## Step 1: GitHub automation

**Ask:** "After each phase, should I auto-run the GitHub flow (branch, commit, pull, push)?"

- If yes → Set `autoGithub: true` and `branchPattern: feat/<slug>` in ONE_MAN_CODE_SETTINGS.md.
- If no → Set `autoGithub: false`.

---

## Step 2: GitHub auth (MCP)

**Check:** Does `.antigravity/mcp.json` (or project/global MCP config) include a GitHub server with credentials?

- If **yes** → "GitHub MCP is configured. Skip to next step?"
- If **no** → Guide user (do not write secrets):
  1. Go to https://github.com/settings/tokens/new
  2. Create a token with `repo` scope (or as required by GitHub MCP).
  3. Add to `.antigravity/mcp.json` under the GitHub server config (see `docs/archive/root-legacy/MCP_SETUP.md` or project MCP docs).
  4. Restart Cursor / reload window.

Then set in ONE_MAN_CODE_SETTINGS.md: `github: enabled` or `disabled` per user choice.

---

## Step 3: Planning flow

**Ask:** "Default planning flow when creating plans: **requirements-first** (behavior first), **design-first** (architecture first), or **quick** (minimal 1–2 phases)?"

- Write to ONE_MAN_CODE_SETTINGS.md: `defaultFlow: requirements-first` | `design-first` | `quick`.

---

## Step 4: MCPs

**List** MCP servers (from `.antigravity/mcp.json` or docs: Prisma-Local, Context7, GitHub, cursor-ide-browser, etc.).

**Ask:** "Which MCPs should be enabled for One Man Code? (list currently installed; user can say enable all / disable X)."

- Write to ONE_MAN_CODE_SETTINGS.md under **MCPs** section: `prisma: enabled`, `context7: enabled`, etc.

---

## Step 5: Skills

**List** skills from `.antigravity/skills/` (e.g. gf-man, gf-guide, gf-planner, gf-security, gf-database, gf-api, gf-mobile, …).

**Ask:** "Which skills should be **disabled** for One Man Code flows? (e.g. gf-mobile if not building mobile)."

- Write to ONE_MAN_CODE_SETTINGS.md: `skillsDisabled: ["gf-mobile"]` or `[]`.

**Rule:** When running `/man run` or `/man ship`, gf-man should skip loading any skill listed in `skillsDisabled`. Do not delete skill files; only skip loading in OMC context.

---

## Template for ONE_MAN_CODE_SETTINGS.md

If the file does not exist, create it with:

```markdown
# One Man Code Settings
Auto-generated by /man settings. Edit manually or re-run /man settings.

## GitHub
- autoGithub: true
- branchPattern: feat/<slug>

## Planning flow
- defaultFlow: requirements-first

## MCPs
- prisma: enabled
- context7: enabled
- github: enabled
- browser: enabled

## Skills
- skillsDisabled: []
```

Then overwrite with wizard results.

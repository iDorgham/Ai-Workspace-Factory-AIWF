## Technique 2 — shadcn/ui Component Architecture

### Component Composition Pattern

```tsx
// Pattern: Extend shadcn primitives, never modify originals
// components/ui/button.tsx stays UNTOUCHED from shadcn
// components/custom/submit-button.tsx extends it:

import { Button, ButtonProps } from '@/components/ui/button';
import { Loader2 } from 'lucide-react';

interface SubmitButtonProps extends ButtonProps {
  isLoading?: boolean;
  loadingText?: string;
}

export function SubmitButton({
  isLoading, loadingText = 'Saving...', children, disabled, ...props
}: SubmitButtonProps) {
  return (
    <Button disabled={disabled || isLoading} {...props}>
      {isLoading ? (
        <>
          <Loader2 className="mr-2 h-4 w-4 animate-spin" />
          {loadingText}
        </>
      ) : children}
    </Button>
  );
}
```

### Component Organization

```markdown
## Directory Structure

components/
├── ui/           # shadcn primitives (NEVER modify directly)
│   ├── button.tsx
│   ├── dialog.tsx
│   ├── input.tsx
│   └── ...
├── custom/       # Your extensions of shadcn primitives

│   └── data-table.tsx
├── layout/       # Page-level layout components
│   ├── header.tsx

    ├── settings/

## Rules:
- ui/* → auto-generated by shadcn CLI, never hand-edit
- custom/* → your wrappers that compose ui/* components
- layout/* → structural components used across pages
- features/* → domain-specific components that combine multiple ui/custom
```

## Technique 3 — Responsive Design System

### Breakpoint Strategy

```css
/* Breakpoint tokens — mobile-first */
/* sm: 640px  — Large phones landscape */
/* md: 768px  — Tablets */
/* lg: 1024px — Small laptops */
/* xl: 1280px — Desktops */
/* 2xl: 1536px — Large screens */

/* Mobile-first: base styles = mobile, then override up */
.container {
  padding-inline: var(--space-4);  /* 16px on mobile */
}

@media (min-width: 768px) {
  .container {
    padding-inline: var(--space-8); /* 32px on tablet+ */
  }
}

@media (min-width: 1280px) {
  .container {
    padding-inline: var(--space-12); /* 48px on desktop */
    max-width: 1280px;
    margin-inline: auto;
  }
}
```

### Responsive Component Pattern

```tsx
// Hook: useMediaQuery for JS-based responsive logic (use sparingly)
'use client';
import { useState, useEffect } from 'react';

export function useMediaQuery(query: string): boolean {
  const [matches, setMatches] = useState(false);
  
  useEffect(() => {
    const media = window.matchMedia(query);
    setMatches(media.matches);
    
    const listener = (e: MediaQueryListEvent) => setMatches(e.matches);
    media.addEventListener('change', listener);
    return () => media.removeEventListener('change', listener);
  }, [query]);
  
  return matches;
}

// Usage
const isMobile = useMediaQuery('(max-width: 768px)');
// Show drawer on mobile, sidebar on desktop
{isMobile ? <MobileDrawer /> : <DesktopSidebar />}
```

## Technique 4 — Next.js Performance Optimization

### Server Component Boundaries

```markdown
## RSC Boundary Rules

SERVER Component (default — prefer this):
  ✅ Data fetching (async/await directly in component)
  ✅ Database queries (Prisma, Drizzle)  
  ✅ Accessing secrets (env vars)
  ✅ Heavy rendering logic
  ✅ Large dependencies that shouldn't ship to client
  → 0KB added to client bundle

CLIENT Component ('use client'):
  ✅ useState, useEffect, useRef (React hooks)
  ✅ Event handlers (onClick, onChange)
  ✅ Browser APIs (localStorage, window, navigator)
  ✅ Interactive libraries (Framer Motion, React Hook Form)
  → Adds to client bundle — minimize scope

## Pattern: Narrow Client Boundaries
// ❌ BAD: Entire page is client component
'use client';
export default function DashboardPage() { /* everything here */ }

// ✅ GOOD: Only interactive parts are client
export default async function DashboardPage() {
  const data = await fetchData(); // Server — no client JS
  return (
    <div>
      <StaticHeader data={data} />     {/* Server — 0KB */}
      <InteractiveChart data={data} /> {/* Client — only chart JS */}
      <DataTable data={data} />        {/* Server — 0KB */}
    </div>
  );
}
```

### Performance Checklist

```markdown
## Next.js Performance Audit

Images:
  □ All images use next/image (automatic WebP, lazy loading, sizing)
  □ LCP image has priority={true}
  □ No layout shift from images (explicit width/height or fill + sizes)

Fonts:
  □ Using next/font (self-hosted, no external request)
  □ font-display: swap configured
  □ Arabic font loaded only for Arabic locale (conditional)

Code splitting:
  □ Dynamic imports for heavy components: dynamic(() => import('./HeavyChart'))
  □ Route groups don't share unnecessary layouts
  □ Third-party scripts use next/script with strategy="lazyOnload"

Data fetching:
  □ Parallel data fetching where possible (Promise.all, not sequential await)
  □ React.cache() for deduplicating same request in render tree
  □ ISR with revalidate for semi-static pages

Bundle analysis:
  □ Run: ANALYZE=true pnpm build (with @next/bundle-analyzer)
  □ No client-side import of server-only libraries (moment.js, lodash full)
  □ Total First Load JS < 100KB (excluding framework chunks)
```

## Anti-Pattern Catalog

| ID | Anti-Pattern | Risk | Fix |
|----|-------------|------|-----|
| FE-001 | Hardcoded hex colors in components | **HIGH** — Breaks dark mode, inconsistent | Use `hsl(var(--token))` CSS variables exclusively |
| FE-002 | Modifying shadcn ui/* files directly | **HIGH** — Breaks on shadcn updates | Extend in custom/* directory; keep ui/* pristine |
| FE-003 | Entire page marked 'use client' | **HIGH** — Ships all code to client, breaks RSC | Narrow client boundary to interactive parts only |
| FE-004 | No next/image for images | **MEDIUM** — Larger files, no lazy loading, CLS | Always use next/image with width/height or fill |
| FE-005 | Desktop-first CSS | **MEDIUM** — Mobile layouts break | Mobile-first: base = mobile, @media min-width for larger |
| FE-006 | Importing full lodash/moment | **HIGH** — 70KB+ added to client bundle | Use date-fns or native Intl; lodash-es with tree-shaking |
| FE-007 | No font preloading | **MEDIUM** — FOUT (Flash of Unstyled Text) | Use next/font with font-display: swap |
| FE-008 | Missing dark mode token for new components | **MEDIUM** — Broken dark mode for new features | Every new CSS variable needs both :root and .dark values |

## Overview

This directory contains the **standard scaffold templates** provisioned into every new workspace during `/compose`. It defines the `docs/` category structure, unified naming conventions, `.gitignore`, and `.env` templates.

---

## Workspace Root Structure (Provisioned by /compose)

```
{workspace-slug}/
├── .gitignore              ← From workspace-scaffold/.gitignore.template
├── .env                    ← From workspace-scaffold/.env.template
├── CLAUDE.md               ← Generated by Factory Scaffolder
├── docs/
│   ├── 00-guides/          ← AIWF-Guide.md + how-tos + /tutorial outputs
│   ├── 01-plans/           ← PRD, requirements, roadmaps
│   ├── 02-specs/           ← spec.md, spec.yaml files
│   ├── 03-architecture/    ← design.md, ERD, DB schema diagrams
│   ├── 04-contracts/       ← API contracts, state contracts
│   ├── 05-reports/         ← Validation reports, audit logs, test results
│   └── 06-releases/        ← Release notes, changelogs
├── plan/
│   ├── 00-prd/
│   └── _archive/
└── .ai/                    ← AIWF intelligence layer (v7 orchestration)
```

## Unified Naming Template

All documents created inside `docs/` follow this convention:

```
{YYYY-MM-DD}_{phase-slug}_{type}.{ext}
```

### Examples
```
docs/01-plans/2026-04-23_00-prd_requirements.md
docs/02-specs/2026-04-23_01-hotel-booking_spec.md
docs/03-architecture/2026-04-23_01-hotel-booking_design.md
docs/03-architecture/2026-04-23_01-hotel-booking_db-schema.sql
docs/04-contracts/2026-04-23_01-hotel-booking_api-contract.yaml
docs/05-reports/2026-04-23_01-hotel-booking_spec-lint-report.md
docs/06-releases/2026-04-23_v1.0.0_release-notes.md
```

### Type Vocabulary (Standardized)
| Type Token | Description |
| :--- | :--- |
| `prd` | Product Requirements Document |
| `requirements` | Requirements trace / REQ-IDs |
| `spec` | Human-readable spec narrative |
| `design` | Architecture and data flow |
| `db-schema` | Database schema SQL |
| `db-erd` | Entity Relationship Diagram |
| `api-contract` | API contract YAML |
| `state-contract` | Application state contract |
| `spec-lint-report` | Spec validation gate output |
| `contract-coverage` | Contract coverage JSON |
| `trace-matrix` | REQ-ID traceability matrix |
| `release-notes` | Release notes markdown |
| `deployment-log` | Deployment audit record |
| `tutorial-session` | /tutorial command output |
| `guide` | How-to or instructional doc |

## Auto-Routing Rule

When any command generates a document, it is auto-routed to the correct `docs/` category using this logic:

```
/do output      → docs/01-plans/{date}_{slug}_prd.md
/plan spec.md   → docs/02-specs/{date}_{slug}_spec.md
/plan design.md → docs/03-architecture/{date}_{slug}_design.md
/plan contracts → docs/04-contracts/{date}_{slug}_{type}.yaml
/test report    → docs/05-reports/{date}_{slug}_{type}.md
/deploy record  → docs/06-releases/{date}_{version}_deployment-log.md
/tutorial       → docs/00-guides/{date}_tutorial-session.md
```

## Overview

The SDD (Spec-Driven Development) Blueprint is the canonical folder structure generated by every `/plan` command. This directory contains template stubs for each file in the blueprint, enabling Factory Scaffolder to provision the full structure deterministically.

---

## Blueprint Folder Structure

```
plan/
├── _manifest.yaml                    # Top-level phase registry
├── 00-prd/
│   ├── prd.md                        # Human-readable PRD with REQ-IDs
│   └── requirements-trace.json       # REQ-ID → phase mapping
├── 01-<phase-slug>/
│   ├── spec.md                       # Human narrative + user flows
│   ├── spec.yaml                     # Machine contract (see spec-schema.yaml)
│   ├── design.md                     # Mermaid architecture + data flow
│   ├── db-schema.sql                 # Database schema
│   ├── db-erd.md                     # ERD with cardinality and FK logic
│   ├── tasks.json                    # Dependency-mapped task graph
│   ├── contracts/
│   │   ├── api-contract.yaml         # REST/GraphQL API contract
│   │   ├── state-contract.json       # Application state contract
│   │   └── test-fixtures/            # One JSON fixture per AC
│   ├── prompts/
│   │   ├── codegen.md                # Code generation prompts
│   │   ├── review.md                 # Code review prompts
│   │   └── debug.md                  # Debug prompts
│   ├── templates/
│   │   ├── boilerplate/              # Starter code templates
│   │   ├── ci-cd/                    # GitHub Actions / Vercel config
│   │   └── logging-standards.md     # Log format standards
│   ├── regional/                     # Only present if --region used
│   │   ├── egypt-compliance.md       # Law 151/2020 checklist
│   │   └── mena-adaptations.json    # Feature flags + payment config
│   └── validation/
│       ├── spec-lint-report.md       # Gate 1: spec.yaml completeness
│       ├── contract-coverage.json    # Gate 2: AC test coverage
│       └── trace-matrix.json         # Gate 3: REQ-ID traceability
└── _archive/                         # Append-only versioned snapshots
```

---

## Template Files

See sibling files in this directory:
- `_manifest.template.yaml` — Top-level manifest stub
- `spec.template.yaml` — spec.yaml stub (see also factory/schema/spec-schema.yaml)
- `tasks.template.json` — tasks.json stub with dependency graph
- `api-contract.template.yaml` — API contract stub
- `trace-matrix.template.json` — Traceability matrix stub

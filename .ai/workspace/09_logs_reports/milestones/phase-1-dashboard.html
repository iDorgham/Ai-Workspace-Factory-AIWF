Command execution flow

Example path for /create blog-posts with fallbacks and versioned outputs.

User: "/create blog-posts" │ ↓ guide-agent parses command │ ├─ Look up in commands-multi-tool.md │ └─ Rank 1: Claude, Rank 2: Gemini, Rank 3: Copilot │ ├─ Load .ai/tool-adapters/claude-adapter.md │ └─ Execute with Claude │ ├─ Success ✓ │ └─ Save: post_1_claude_v1.md │ ├─ Timeout (>300s) │ └─ Try Gemini (Rank 2) │ └─ Low quality (orig < 15%) └─ Retry with Gemini (different prompt) │ ├─ Success ✓ │ └─ Save: post_1_gemini_v1.md │ └─ Fail └─ Try Copilot (Rank 3) │ ├─ Success ✓ │ └─ Save: post_1_copilot_v1.md │ └─ Fail ✗ ERROR └─ All tools exhausted Manual intervention needed

State management

Global session state plus per-tool files and performance logs.

Global state

Shared by all tools

File: .ai/memory/state.json
Tracks: Pipeline stage

Access: All tools (read/write)

Tool-specific state

One file per tool (Phase 1)

Location: .ai/memory/multi-tool-state/
Files: claude.json, gemini.json
Tracks: Tokens, cost, history
Access: Tool-specific only

Performance log

Tool metrics for ranking updates

Records: Latency, cost, quality
Frequency: Every command

Phase 1 activation checklist

Three steps to go from “files on disk” to monitored production use.

Before you start: Infrastructure files are in place. Complete the steps in order and verify logs after each smoke test.

Step 1 — Update guide-agent system prompt

File: .ai/GUIDE-AGENT-SYSTEM-PROMPT-ADDENDUM.md

Copy the section labeled “LOAD THIS SECTION: Multi-Tool Orchestration (Phase 1)” into the guide-agent system prompt.

Time: about 2 minutes

Step 2 — Run smoke tests

Test 1 — Tool selection

/create blog-posts about interior design

Expected: Claude selected; outputs use a _claude_v1.md suffix pattern.

Test 2 — Multimodal (Gemini)

/optimize images in content/

Expected: Gemini selected; results reflected in assets-seo.json.

Test 3 — State sync

cat .ai/memory/state.json | grep "last_tool"

Expected: "last_tool": "claude" or "gemini" after runs.

Time: about 5 minutes

Step 3 — Monitor for one week

Watch

  • Tool success rate (target > 95%)
  • Cost vs. expectations (rough bands: Claude ~$0.25, Gemini ~$0.08 per relevant window)
  • Latency (Claude ~800ms, Gemini ~1200ms illustrative)
  • Log continuity (tool field present, no silent failures)

Files

  • logs/workflow.jsonl
  • logs/tool-performance.jsonl
  • .ai/memory/multi-tool-state/

Time: about 5 minutes per day

Ready: All referenced paths exist in the workspace layout. You can begin activation whenever the prompt merge is approved.

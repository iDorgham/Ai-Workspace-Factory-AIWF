# `/factory help`

## Syntax
`/factory help`

## Required Inputs
- Client/profile arguments as specified in syntax.

## Outputs
- Terminal status summary.
- Updated files under `factory/intake`, `factory/manifests`, `workspaces/<slug>`, or `factory/reports` depending on command.

## Failure Conditions
- Missing profile, invalid intake data, malformed library metadata, dependency mismatch, or validation failure.

## Beginner Example
`/factory help`

## Purpose
Display factory command quick reference and examples.

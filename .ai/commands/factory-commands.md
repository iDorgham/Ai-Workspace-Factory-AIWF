## 🏗️ 1. Composition & Workspace Commands (`/factory`)

### `/factory intake [client]`
- **Purpose**: Run guided discovery and persist client intake JSON.
- **Usage**: `/factory intake acme`
- **Impact**: Initializes the project lifecycle in `factory/intake/`.

### `/factory profile [list|show]`
- **`list`**: List all available workspace composition profiles.
- **`show [name]`**: Display required/optional modules and overrides for a specific profile.
- **Examples**: 
    - `/factory profile list`
    - `/factory profile show fullstack-saas`

### `/factory validate [client]`
- **Purpose**: Run required structural and dependency validation checks.
- **Usage**: `/factory validate acme`
- **Output**: Generates a validation report in `factory/reports/`.

### `/factory compose [client] [--profile]`
- **Purpose**: Compose the workspace component set and produce a composition manifest.
- **Usage**: `/factory compose acme --profile fullstack-saas`
- **Impact**: Finalizes the blueprint before materialization.

### `/factory generate [client]`
- **Purpose**: Compose then materialize the generated workspace output.
- **Usage**: `/factory generate acme`
- **Impact**: Scaffolds the full project in `workspaces/<client>/`.

### `/factory diff [client]`
- **Purpose**: Compare the generated workspace against manifest expectations to detect drift.
- **Usage**: `/factory diff acme`

## 📚 2. Library & Maintenance Commands (`/factory`)

### `/factory library [status|refresh|search]`
- **`status`**: Show indexed component counts and last sync times.
- **`refresh`**: Regenerates indexes and validates metadata (use after manual library updates).
- **`search [type] [tag]`**: Semantic search of indexed items by type (agents, skills, etc.) and tag.
- **Examples**:
    - `/factory library status`
    - `/factory library refresh`
    - `/factory library search skills seo`

### `/factory doctor`

- **Usage**: `/factory doctor`

### `/factory help`

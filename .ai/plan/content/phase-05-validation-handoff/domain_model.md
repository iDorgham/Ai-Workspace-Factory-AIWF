## 1. Validation Framework

### KPI Checkpoint Architecture

| Checkpoint | Day | Metrics | Decision |
|------------|-----|---------|----------|
| Early signal | 7 | GitHub stars delta, LinkedIn impressions C-02, X thread engagement C-03 | Adjust Day 10+ tone if signal weak |

## 2. Content Audit Model

After Day 30, each published piece is audited against:

| Dimension | Metric | Target |
|-----------|--------|--------|
| SEO performance | Keyword rank delta | Primary keyword ranking improved |
| Engagement rate | Likes + shares / impressions | LinkedIn ≥3%, X ≥2% |
| Conversion | GitHub link clicks / impressions | ≥1.5% |
| Brand voice | Post-publish reader feedback (DMs) | No "hype" or "vague" complaints |
| MENA performance | Arabic piece engagement vs EN equivalent | Within 30% |

---

## 3. Retro Framework (4 Questions)

1. **What worked?** — Which pieces over-indexed on KPIs? What made them work?
2. **What didn't?** — Which pieces under-indexed? Was it the brief, the generation, or the distribution?
3. **What do we know now about personas?** — Did actual audience match modelled personas?
4. **What changes for Phase 06 content plan?** — Pillar adjustments, channel deprioritisation, new pieces
